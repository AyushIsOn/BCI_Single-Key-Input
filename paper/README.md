# Flow Dial — India HCI 2026 paper source

ACM `acmart` source for the submission, in the **single-column review format**
India HCI requires.

## Files
- `flow_dial.tex` — the paper (uses `\documentclass[manuscript,review,anonymous]{acmart}`)
- `flow_dial.bib` — references (Dasher, word-gesture keyboards, BCI typing, keyboard modeling)
- `figures/` — `fig_wpm_vs_noise.png`, `fig_error_vs_noise.png` (generated by `../flow_eval/experiment.py`)

## Build on Overleaf (recommended)
1. Create a new project and upload `flow_dial.tex`, `flow_dial.bib`, and the `figures/` folder.
2. Set the compiler to **pdfLaTeX**.
3. Compile. The `[review]` option adds line numbers; `[anonymous]` hides author identity for review.

## Build locally
```bash
pdflatex flow_dial
bibtex   flow_dial
pdflatex flow_dial
pdflatex flow_dial
```
(Requires a TeX distribution with the `acmart` class and `ACM-Reference-Format` style.)

## Format notes
- For the camera-ready 2-column version, switch the first line to
  `\documentclass[sigconf]{acmart}` and fill in author/rights metadata.
- Target length per the CFP is ~7,000–8,000 words excluding references and
  captions; this draft is well within that and can be expanded with the
  surrogate-cursor pilot results once collected.

## Authorship / AI-use note
The India HCI CFP discourages AI content generation and asks that the ideas be
the authors' own. This repository was produced as an engineering/analysis aid:
the simulation, prototype, and figures are reproducible artifacts. **Before
submitting, the authors must review, verify, and rewrite the prose in their own
voice, confirm every cited reference, and take full responsibility for the
content.** Treat this `.tex` as a structured draft scaffold, not a final
author-authored manuscript.
